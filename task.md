# План миграции проекта на NestJS и Nuxt 4

## Статус: Выполнено ✅

Проект успешно мигрирован на современный стек технологий. Основные цели достигнуты: бэкенд перенесен на NestJS, фронтенд на Nuxt 4, база данных на SQLite с использованием Prisma.

### 1. Бэкенд (NestJS) ✅
- [x] Инициализация проекта и настройка Fastify.
- [x] Настройка Prisma и SQLite.
- [x] Реализация модулей:
  - `Auth`: JWT аутентификация, проверка Telegram initData.
  - `Users`: управление пользователями.
  - `Projects` (ранее `Blogs`): создание и управление проектами.
  - `Channels`: интеграция с соцсетями (Telegram, Instagram и др.).
  - `Posts`: создание и планирование постов.
- [x] Настройка отдачи статики (Nuxt build) через Fastify.
- [x] Реализация SPA-роутинга через глобальный фильтр исключений.

### 2. Фронтенд (Nuxt 4) ✅
- [x] Инициализация Nuxt 4 и настройка `nuxt.config.ts`.
- [x] Интеграция Pinia для управления состоянием.
- [x] Реализация API-клиента (`useApi`) с поддержкой JWT.
- [x] Миграция страниц и компонентов из старого проекта (`_old`).
- [x] Настройка i18n и FormKit.
- [x] Перевод ключевых страниц на новый API (camelCase).

### 3. Инфраструктура и Сборка ✅
- [x] Переход на ESM.
- [x] Настройка build-скриптов.
- [x] Решение проблем с Dependency Injection в NestJS.

## Следующие шаги
1.  **Сборка проекта**:
    ```bash
    cd ui && npm install && npm run build
    cd ..
    npm install && npm run build
    ```
2.  **Запуск**:
    ```bash
    node dist/src/main.js
    ```
3.  **Очистка**: После подтверждения работоспособности можно удалить папку `_old`.
